# REQ-700: アプリケーション診断ログ

> **作成日**: 2026-04-13
> **対応Issue**: #24
> **ステータス**: 確定

---

## 要求

障害発生時にアプリケーションの内部動作を追跡し、バグの原因を特定できるようにする

## 理由

バグのないアプリは存在しない。利用者から「おかしい」と報告があったとき、operation_logs（業務操作ログ）だけでは「何をしたか」はわかるが「なぜ失敗したか」がわからない。エラー箇所・入力データ・モジュール名を記録するファイルベースの診断ログが必要

---

## REQ-701: アプリケーション診断ログの記録

アプリケーションの全Rust層（CMD/BIZ/IO/MNT）の処理状況をファイルに記録する

### ログレベル定義

| レベル | 記録する場面 | 通常運用 |
|---|---|---|
| ERROR | 処理が失敗して結果を返せなかった（CMD層のエラー境界で1回記録） | 記録する |
| WARN | 処理は続行したが異常な状態（在庫マイナス、棚卸し中の操作等。BIZ層で記録） | 記録する |
| INFO | 主要な処理の開始/完了（DB初期化完了、CSV取込み完了、バックアップ完了等） | 記録する |
| DEBUG | 内部の判断分岐、SQLクエリ詳細、IO層のエラー詳細 | 開発時・トラブル調査時のみ |

### ログ出力の層別ルール（重複防止）

- **CMD層**: エラー境界として `ERROR` を出力。フロントエンドに返すエラーの最終記録地点
- **BIZ層**: 業務判断に関わる異常を `WARN` で出力。主要処理の完了を `INFO` で出力
- **IO層**: SQLクエリ失敗等の詳細を `DEBUG` で出力
- **MNT層**: バックアップ・マイグレーション等の完了/失敗を `INFO` / `ERROR` で出力

### ログエントリに含める情報

- タイムスタンプ（ISO 8601、ローカル時刻）
- ログレベル
- モジュール名（例: `biz::csv_import_service::commit`）
- メッセージ（処理中のデータ識別子を含む: product_code, csv_import_id 等）

---

## REQ-702: ログファイルの管理

- **保存先**: アプリデータフォルダ/logs/ ディレクトリ
- **ファイルローテーション**: 日次で新ファイル（例: `app.2026-04-13`）。tracing-appenderの命名規則に準拠
- **保持期間**: 30日。超過したログファイルはアプリ起動時に自動削除（1日1回チェック、MNT-02のlog_cleanupと同じパターン）
- **起動順序**: ログ初期化はアプリ起動シーケンスの最初に行う。DB初期化・マイグレーションより前に実行し、起動シーケンス中の障害もログに記録できるようにする
- **初期化失敗時**: ログ初期化自体が失敗した場合（ディスク容量不足、ディレクトリ権限問題等）はstderrに出力してアプリ起動を続行する

---

## REQ-703: エラー発生時の詳細記録

ERRORレベルのログには以下を含める:
- モジュール名（例: `biz::csv_import_service`）
- 処理中のデータ識別子（product_code, csv_import_id等）
- エラー内容（BizError / DbError のvariant + メッセージ）

CMD層でエラーをフロントエンドに返す際に、自動的にERRORログを出力する。BIZ層・IO層ではエラーを上位に伝搬するのみとし、ERRORの重複記録を防ぐ。

---

## REQ-704: ログレベルの動的変更（将来拡張）

- app_settingsからログレベルを変更できる
- **優先度: 低**。初期実装では固定レベル（INFO）でリリースし、運用段階でログ量が問題になった場合に後から追加

---

## REQ-705: ログファイルへのアクセス手段（将来拡張）

- 設定画面に「ログフォルダを開く」ボタンを設置し、利用者がログファイルを開発者に送付できるようにする
- **優先度: 低**。第6段階のUI-11（設定画面）実装時に合わせて追加可能

---

## operation_logsとの関係

| | operation_logs (MNT-02) | 診断ログ (本要求) |
|---|---|---|
| 目的 | 利用者が「いつ何をしたか」を確認 | 開発者がバグの原因を特定 |
| 読む人 | 利用者（設定画面） | 開発者（ログファイル） |
| 保存先 | SQLite (operation_logsテーブル) | ファイル (app_data/logs/) |
| 保持期間 | 365日 | 30日 |
| 記録内容 | 業務操作の成功記録 | エラー箇所・入力データ・モジュール名 |

両者は**独立して共存**する。operation_logsは現行設計のまま維持する。
