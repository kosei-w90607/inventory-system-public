## 設計判断 A〜G

### A. `list_movements` specta 化のタイミング
- **採用**: 本 PR では `get_stock_detail` のみ specta 化、`list_movements` は Phase 4 UI-06c 着手時に specta 化
- **根拠**: (1) 本 PR では `list_movements` 未使用 (遷移リンク disabled)、(2) `MovementQuery`/`MovementRecord` 型は 7+ field の nested 構造で specta 化付随影響 (rename_all + Type derive + Vec 派生型) があり scope 肥大化、(3) PR #66 H-5 で `SalesReportType` 単一 enum specta 化と本件は同質ではない
- **却下案**: 本 PR で両方 specta 化 → **却下理由**: scope 肥大化 + bindings 行数増 + 未使用型を bindings に乗せる drift リスク

### B. 詳細カード展開 UI
- **採用**: **テーブル下部固定の詳細カード** (商品リスト行クリックで一覧テーブル下部に詳細カードを collapsible 展開表示、`src/components/ui/collapsible.tsx` を新規 add で利用。list query 分岐の外に独立配置し list 失敗時も描画 = 部分障害許容と整合。当初案を Codex Round 1 で部分障害許容契約との衝突により変更、経緯は `docs/function-design/58-ui-stock-inquiry.md` §58.7 更新履歴)
- **実装方針** (Round 1 minor 2 反映): Radix Collapsible primitive は `package.json:35` `radix-ui: ^1.4.3` umbrella package の transitive dep として既存、**npm install 不要**。`src/components/ui/accordion.tsx` の pattern (`import { Accordion as AccordionPrimitive } from "radix-ui"`) を踏襲し、commit 5 で `src/components/ui/collapsible.tsx` を**純 TS file 追加のみで新規生成** (shadcn add コマンド経由ではない、npm install 凍結遵守)
- **根拠**: (1) 検索結果一覧と詳細を並列確認したい用途 (発注判断)、(2) Dialog はキーボード trap で「次の商品」動線を遮る、(3) SCREEN_DESIGN.md L126 「商品クリックで詳細カードが展開」と「展開」表現に合致
- **却下案 1**: Dialog → **却下**: 並列確認不可、利用者デモで「次の商品見たい」が 2 ステップに
- **却下案 2**: Resizable 2 ペイン → **却下**: SCREEN_DESIGN.md 文言と乖離、新規依存検討要 (npm install 凍結中)

### C. 「在庫切れ」「在庫少」チップ時の検索ボックス・部門フィルタ挙動

### H. 在庫状態色分け契約 (Round 4 user P2-3 反映、SCREEN_DESIGN.md:129 「在庫ゼロは赤文字」遵拠必須)

### I. 「すべて」チップの検索駆動表示 (Round 4 user P2-5 反映、ラベルと実データの一致)
