# Test Design Matrix: UI-01b 商品登録・修正 画面デザイン polish

## Risk

Risk: R3

## Contracts Under Test

- SPEC-UI-01B-POLISH-DH: UI-01a / UI-01b / UI-06a の画面タイトル h1 統一。
- UI-01a-D8: 一覧の廃番状態は商品名セル内 text badge + 行 muted、専用状態列なし。
- UI-01b-D10: フォーム 4 セクション分割と h2 見出し。
- UI-01b-D11: read-only 入力は readOnly + bg-muted、数量単位 select は disabled 維持。
- UI-01b-D12: 必須項目ラベルに（必須）、色符号化しない。
- UI-01b-D13: 廃番にする操作のみ確認ダイアログ、表示に戻すは直接実行。
- UI-01b-D14: 保存成功 toast、id 重複なし。

## Failure Modes

- 画面タイトルが画面ごとに別サイズで、視覚言語が分裂する。
- 廃番表示が色だけで符号化され、状態列削除で値が消える。
- read-only 入力が disabled の opacity-50 で読めない、または編集可能に見える。
- 数量単位 select が readonly 化されて操作不能になる。
- 必須項目が色だけで示され、（必須）テキストが欠ける。
- 廃番にする操作で確認なしに状態が変わる、または「表示に戻す」が余計な確認を出す。
- 保存成功が無通知、または toast id が重複して積み重なる。

## Negative Paths

- 廃番ダイアログのキャンセル / Esc で `toggleDiscontinue` が呼ばれない。
- 「表示に戻す」操作はダイアログを出さず直接実行する。
- read-only 入力への変更が値に反映されない。
- 数量単位 select は readonly 化されず disabled のまま。

## Boundary Checks

- heading role / text: 3 画面の h1 が role=heading で取得できる。
- 廃番状態: 商品名セル内 text badge + 行 muted。
- readonly 属性: JAN / 現在庫。
- disabled 属性: 数量単位 select。
- 必須ラベル: 部門 / 売価 / 原価 / 商品名（regex 一致）。
- toast id: `product-save-success`。

## Compatibility Checks

- UI-01a route / search params が有効なまま。
- UI-01b の create / edit 保存・廃番フローが通る。
- 既存 backend テストは変更しない。
- generated command 名（`createProduct` / `updateProduct` / `toggleDiscontinue` / `listSuppliers`）は変更しない。
- DB schema / migration 変更なし。

## Data Safety Checks

- 合成データのみ。
- `.env*`、ローカル DB、POS / 店舗 artifacts、backups、logs、receipt images を commit しない。
- destructive command なし。

## Main Wiring / Integration Checks

- route tree generation / typecheck / build。
- `cargo run --bin generate_traceability -- --check`。
- `bash scripts/doc-consistency-check.sh`。
- UI-01a `商品登録` / `修正` の新 route 導線が維持される。

## Residual Test Gaps

- Windows native UI-01b L3 は手動 evidence で、自動化しない。
- 一覧の部門列削除は scope 外（L3 で密度過多なら次候補）。
- PageHeader 共通 component 抽出は rule-of-three を満たした時点で別 PR とする。
