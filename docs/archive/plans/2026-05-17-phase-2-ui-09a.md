### 2.4 単価派生計算仕様 (user Option 1.5 確定、Round 2 β-2 修正)

```ts
// src/features/daily-sales/lib/calculate-unit-price.ts

/**
 * 売上明細の実績単価を計算する純関数。
 * 商品マスタの販売単価ではなく、売上記録の金額 ÷ 数量で求めた派生値。
 * 返品行 (quantity<0 + amount<0) では絶対値で「単価の大きさ」として表示。
 * quantity=0 の場合は null を返し、表示側で「—」placeholder。
 * 端数は四捨五入で整数円表示 (Math.round)。
 */
export function calculateEffectiveUnitPrice(item: DailySaleItem): number | null {
  if (item.quantity === 0) return null;
  return Math.round(Math.abs(item.amount) / Math.abs(item.quantity));
}
```

**テストケース (5 ケース、user 提案準拠)**:
1. `amount=2376, quantity=4` → 594
2. `amount=-594, quantity=-1` → 594 (返品行)
3. `amount=2002, quantity=2` → 1001
4. `amount=1000, quantity=3` → 333 (四捨五入)
5. `quantity=0` → null

**ソート挙動**:
- 列ヘッダ click で `sortBy: "unit_price"` → URL 反映、`sortDailyItems(items, "unit_price", direction)` が `calculateEffectiveUnitPrice(item)` を計算してから数値比較
- `null` (quantity=0 行) は **末尾配置** (asc/desc 共通、user 提案準拠)

## 4. URL state 設計 (zod 4 直接渡し、sortBy 5 列対応、Round 2 β-2 修正)

```ts
// src/routes/reports/daily.tsx
import { createFileRoute } from "@tanstack/react-router";
import { z } from "zod";
import { DailySalesPage } from "@/features/daily-sales/DailySalesPage";

const searchSchema = z.object({
  date: z.string().regex(/^\d{4}-\d{2}-\d{2}$/).optional().catch(undefined),
  dept: z.coerce.number().int().positive().optional().catch(undefined),
  sortBy: z.enum(["product_code", "name", "quantity", "unit_price", "amount"]).optional().catch(undefined),
  //                                              ^^^^^^^^^^^^ Round 2 β-2 追加
  sortDir: z.enum(["asc", "desc"]).optional().catch(undefined),
});

export type SearchParams = z.output<typeof searchSchema>;  // Round 2 β-4 修正

export const Route = createFileRoute("/reports/daily")({
  validateSearch: searchSchema,  // zod v4 直接 (context7 公式推奨)
  component: DailySalesPage,
});
```

### 4.1 デフォルト値とフォールバック
- `date` 未指定 → 当日 (`useTodayDate()`、§2.7 で daily-sales 内閉鎖)
- `dept` 未指定 → `null` = 「すべて」
- `sortBy` 未指定 → `null` = ソートなし (BIZ 層の `department_id` 昇順を維持)
- `sortDir` 未指定 → `asc` (sortBy 指定時のみ意味あり)

### 4.2 不正値の挙動
zod v4 で `?date=invalid` / `?dept=abc` / `?sortBy=garbage` 等が来た場合:
- `.optional().catch(undefined)` で各 field 単位に undefined fallback
- schema 全体は throw しない設計
- 手動検証項目 §7 第 11 項目で「URL 直接編集で全 field undefined fallback → 当日 + 全部門 + ソートなし表示で error 出ない」を確認

### 4.3 search params 部分更新

```ts
const search = Route.useSearch();  // 型推論 = SearchParams (z.output<typeof searchSchema>)

  navigate({ search: (prev) => ({ ...prev, date: newDate }) });
```

## 5. TanStack Query 設計

### 5.1 staleTime 統合方針 (UI-00 と 5min 統一)
- 当日 useQuery: `staleTime: 5 * 60_000`、gcTime 10min
- 前日 useQuery: `staleTime: 5 * 60_000`、gcTime 10min
- UI-00 と同 queryKey に異なる staleTime の anti-pattern を回避
- 動的 staleTime 案 (当日 30sec / 過去日 24h) は §9 代替案で却下根拠

### 5.2 前日 useQuery の enabled 条件
**前日 useQuery は常時 `enabled: true`** (UI-00 の部分障害許容パターン 53-ui-home.md D-3 と整合):
- 当日 fail 時 = 画面全体 Alert (`<Alert variant="destructive">` + 再試行ボタン)、前日比カードは表示されない
- 前日 fail 時 = 前日比カードのみ「比較データなし」表示、他 3 カード + テーブル正常
- 当日成功 + 前日 0 円 = 「比較不可」表示 (除算回避)
- 当日成功 + 前日成功 = 通常の前日比表示

### 5.3 export useMutation + Sonner dedup (Round 2 δ-1 修正)
- queryKey なし (副作用なし、cache invalidate しない)
- 成功時: `toast.success("CSV を保存しました (${N} 件)", { id: "export-daily-csv-success" })` ← id-based dedup、UI-00 PR #56 P2-B horizontal pattern
- 失敗時: `toast.error("CSV 出力に失敗しました: ${message}", { id: "export-daily-csv-error" })`

### 5.4 UI-07 commit/rollback の invalidation chain (Round 2 β-3 修正、本 PR scope に格上げ + Round 3 B-2 反映)
**取込み直後の主動線 (UI-07 → UI-09a) で必須のため、本 PR scope 内で対応**:
- `src/features/csv-import/hooks/useCsvImportFlow.ts:113-122` の commit/rollback success handler の現状 = `csvImportLists()` + `dailySales(settlementDate)` (単一 date) + `lowStock()` + `pluDirty()` の 4 invalidate
- 本 PR で `dailySales(settlementDate)` 単一 invalidate を **`["daily-sales"]` prefix invalidate に置換** (二重 invalidate 防止、Round 3 B-2 判断)
- 置換後の chain: `csvImportLists()` + `["daily-sales"]` prefix (= 全 date refetch) + `lowStock()` + `pluDirty()` の 4 invalidate
- staleTime 5min を超える前に CSV 取込み完了 → UI-09a 遷移時に最新データ表示
- **副次効果** (Round 3 B-1): `["daily-sales"]` prefix invalidate は `dailySales(yesterday)` (UI-00 ホーム) + `dailySales(today)` (UI-09a) 両方が refetch、UI-00 ホームの SummaryCards `useHomeSummary.ts:19-29` の `sales useQuery` も波及して最新表示、これは取込み直後 UX 上望ましい
- 同時に **UI-07 関数設計書 `55-ui-csv-import.md` §55.3 invalidation 設計** も sync 更新 (Round 3 B-2): csvImportLists prefix のみ記述だった箇所を「+ `["daily-sales"]` prefix (UI-09a + UI-00 ホーム両方への波及)」追記

### 5.5 useExportDailySalesCsv の Blob memory 安全性 (Round 2 γ-6 修正)
- `URL.revokeObjectURL(url)` のタイミング = `setTimeout(() => URL.revokeObjectURL(url), 100)` で Chromium download 中断回避
