## 1. Context

### なぜこの PR が必要か

PR #44 (Prettier) / PR #45 (ESLint + typecheck 独立化) で「道具立て」は揃った。しかし以下 3 点が未解決:

- **pre-commit hook が未導入**: `.husky/` も `lefthook.yml` もなく、`scripts/pre-push.sh` は Rust + 設計書のみ担当で frontend は手動運用
- **CI frontend ジョブが `typecheck + build` のみ**: PR #45 で整備した `npm run lint` / `npm run format:check` の regression gate が CI に無い
- **npm audit 3 vulnerabilities が放置**: moderate × 2 (smol-toml via markdownlint-cli2), high × 1 (vite 7.0.4 関連 3 CVE) の可視化経路なし

PR C でまとめて解消し、Phase B-1 を閉じる。

### Definition of Done

1. `npx lefthook install` 後、`*.{ts,tsx,js,jsx}` を staged した状態で `git commit` すると staged ファイルのみに `eslint --fix` + `prettier --write` が走る
2. `*.{json,md,yml,yaml}` の staged 時は `prettier --write` のみ走る
3. `git commit --no-verify` で bypass 可能（緊急時用）。`merge` / `rebase` 中は自動 skip
4. CI frontend ジョブが `typecheck → lint → format:check → build → npm audit` の順で実行
5. `npm audit --audit-level=high` step は exit ≠0 でも CI を落とさない（warn-only）
6. `docs/DEV_SETUP_CHECKLIST.md` §3 に lefthook install 手順が追記される
7. `Plans.md` の Phase B-1 進捗が「PR C ✅」に更新され、次着手が 7-5c に戻る
