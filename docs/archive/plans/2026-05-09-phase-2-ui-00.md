## specta 化作業（バックエンド）

**対象 4 コマンド**:
- `src-tauri/src/cmd/sales_cmd.rs::get_daily_sales`（引数: `date: String`）

**必要作業**:
1. 4 コマンド関数に `#[specta::specta]` 属性付与
2. 戻り値型に `#[derive(specta::Type)]` 付与:
   - `DailySalesReport` / `GrandTotal` / `DailySaleItem` / `DeptSubtotal`（`src-tauri/src/biz/sales_service.rs`）
   - `ProductResponse`（plu_export_cmd.rs 近傍）
   - `ProductWithRelations`（bindings.ts L93-96 既存、確認のみ）
   - `CsvImport`（`src-tauri/src/db/sales_repo.rs` 近傍）
   - `PaginatedResult<T>`（bindings.ts L37-42 既存、`CsvImport` に対しても generic 展開されるか確認）
3. 推移参照型の確認: 上記戻り値型が参照する nested struct / enum / `chrono::NaiveDate` 相当の型変換（`DbConnection` の Local 扱い含む）を `src-tauri/src/` 横串で確認、漏れあれば追加 derive
4. `src-tauri/src/lib.rs::export_specta_bindings()` 内の `collect_commands!` マクロに 4 コマンド追加（L40-43 既存 2 行に追記）
5. **bindings 再生成**: `cargo run --bin generate_bindings`（bin 実在確認済: `src-tauri/src/bin/generate_bindings.rs`）→ `src/lib/bindings.ts` 再生成
6. ローカルで `git diff src/lib/bindings.ts` で型形状確認:
   - 引数が camelCase 化されていること（D-7）
   - 既存 `commands.searchProducts` / `commands.getProduct` の型が壊れていないこと
   - `PaginatedResult<CsvImport>` の generic 展開が想定通りか
   - String 型フィールド（`source` 等）が TS `string` に縮退している場合、本 PR では修正しない（D-10、Phase 3 に送る）
7. CI drift check: `.github/workflows/ci.yml` で `cargo run --bin generate_bindings && git diff --exit-code` が走る（PR #48 commit `126e6fb` で追加済）。commit 漏れで CI fail

**Exit 条件**（別 PR に切り替える判定）: specta 化が破壊的変更を誘発する場合（既存 `commands.searchProducts` / `getProduct` の型が壊れる、あるいは Rust 側で serde rename が必要になる等）。通常は同 PR 完結。

P2-A 対応で hooks と components を 1 コミットに統合（hooks 単独コミットで使用側なし状態を回避）。pre-push hook は push 時に HEAD 状態で走るが、レビュー可読性のため使用側が同時に揃う形にする。
