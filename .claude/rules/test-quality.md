# テスト品質保護

## 禁止事項

- 既存のテストを削除しない
- `#[ignore]` や `#[cfg(skip)]` でテストを無効化しない
- テスト内の assert を削除・弱体化しない（例: `assert_eq!` → `assert!(true)` への書き換え）
- テストを通すために本番コードの仕様を変えない（テストが正しい場合）

## 必須事項（テストルール）

- IMPORTANT: 関数実装とテストは必ず同時に書く。後でまとめて書くことは禁止
- テスト関数名に REQ 番号を含める: `test_create_product_req101_normal`
- テスト内にコメントで仕様IDを記載: `// REQ-101-01: JANコード入力で商品登録`
- 新規テストは `cargo test` で全パスを確認してからコミット

## テスト失敗時の対応

1. まずテストが正しいか確認する（設計ドキュメントと突合）
2. テストが正しければ本番コードを修正する
3. テストが間違っていれば理由をコメントに残して修正する
